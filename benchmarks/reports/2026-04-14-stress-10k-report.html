Test Methodology

Tool: k6 v1.7.1 · Transport: HTTP/1.1 over TCP loopback (127.0.0.1) · Connection: Keep-alive ON (connections reused, same as competitors)
Hardware: Apple Silicon M-series · Build: Rust release (optimised, LTO) · Version: ThinkingRoot v0.9.0

Load profile:
0→500 VUs (1m) → 500→2K (2m) → 2K→5K (3m) → 5K→10K (3m) → sustain 10K (5m) → drain (2m)

Endpoint under test: GET /api/v1/ws/{workspace}/entities — pure in-memory HashMap read, no database I/O, no network calls beyond the HTTP stack.

Competitor benchmarks: FalkorDB p50 from their 2024 blog; SuperMemory p95 from their documentation; Zep latency from their GitHub README; Graphiti from published paper. All competitor tests were run at ≤50 VUs on remote SaaS endpoints.

What this proves: ThinkingRoot's REST API, embedded on the same machine as the AI agent, delivers knowledge retrieval at sub-10ms p95 even under 10,000 concurrent connections — without any remote database, vector store, or LLM in the read path.
